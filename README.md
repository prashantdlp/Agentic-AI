# Agentic AI Learning Journey

## Overview

This repository documents my learning journey in **Agentic AI**, where I explored the concepts behind autonomous AI systems and built practical projects using modern LLM frameworks.

The course progressed from prompt engineering fundamentals to building multi-step AI agents capable of reasoning, planning, and interacting with a development environment.

---

## Learning Timeline

### Week 1 — Prompt Engineering

Focused on understanding how Large Language Models respond to different prompting strategies.

Topics covered:

* Basics of Large Language Models (LLMs)
* Prompt Engineering
* Zero-shot Prompting
* Few-shot Prompting
* Prompt Design Best Practices
* Role-based Prompting

**Key Takeaway:**
Well-designed prompts significantly improve the quality, consistency, and reasoning ability of LLM outputs.

---

### Week 2 — Chunking & Retrieval-Augmented Generation (RAG)

Learned how to provide external knowledge to LLMs using retrieval systems.

Topics covered:

* Why LLMs need external knowledge
* Document Chunking Strategies
* Text Splitters
* Embeddings
* Vector Databases
* Similarity Search
* Retrieval-Augmented Generation (RAG)

**Key Takeaway:**
Instead of relying solely on model memory, RAG enables LLMs to answer questions using relevant external documents.

---

### Week 3 — LangChain & Agent Development

Introduced frameworks for building AI agents capable of reasoning and tool usage.

Topics covered:

* LangChain
* LangGraph
* Chains
* Prompt Templates
* Output Parsers
* Tool Calling
* Agents
* Memory
* Multi-step Reasoning
* Environment Interaction

**Key Takeaway:**
LangChain/LangGraph simplifies building intelligent applications by combining LLMs with tools, memory, and workflows.

## Project — Development Environment Assistant

### Description

The main project developed during the course is a **Development Environment Assistant**, an Agentic AI system that helps developers diagnose and manage their local development environment.

Unlike a traditional chatbot, the assistant can interact with the operating system through controlled Bash command execution, allowing it to inspect the environment and provide accurate, real-time responses.

### Features

* Execute Bash commands safely
* Inspect installed software and package versions
* Check Python, Node.js, npm, Git, Docker, and other development tools
* Diagnose missing dependencies
* Detect version mismatches
* Verify environment configurations
* Provide troubleshooting suggestions
* Maintain conversational context while solving issues

### Example Queries

* What version of Python do I have?
* Is Docker installed?
* Which Node.js version is currently active?
* Do I have Git configured correctly?
* Check whether PostgreSQL is running.
* Show all installed Python packages.
* Verify my development environment for this project.
* create a venv environment and install numpy in that venv.
